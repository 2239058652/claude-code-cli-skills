## Brainstorming (头脑风暴，直接在 /bs 后输入问题)

Before any implementation work, you MUST:

1. Explore project context (files, docs)
2. Ask clarifying questions ONE at a time — purpose, constraints, success criteria
3. Propose 2-3 approaches with trade-offs and a recommendation
4. Present the design in sections, get approval after each section
5. Only then move to implementation

Key rules:
- Do NOT write any code before design is approved
- One question per message
- YAGNI: remove unnecessary features ruthlessly
- Every project goes through this — no exceptions for "simple" tasks
